# БюджетPRO — Полное руководство: превращаем в нативное Android-приложение
## Linux Mint | Для начинающих | Пошагово

---

## ЧТО МЫ ДЕЛАЕМ И ПОЧЕМУ

Ваш проект — это веб-сайт (HTML + CSS + JS). Чтобы он стал настоящим Android-приложением (.apk файл), мы используем инструмент **Capacitor** от компании Ionic. Он "оборачивает" сайт в нативную оболочку — получается полноценное приложение, которое:
- Устанавливается как обычное приложение из файла .apk
- Работает полностью офлайн
- Имеет иконку на рабочем столе
- Не показывает адресную строку браузера
- Имеет доступ к биометрии, уведомлениям, файловой системе

## ЧАСТЬ 1 — ПОДГОТОВКА КОМПЬЮТЕРА

### Шаг 1.1 — Установка Node.js

Node.js — это среда для запуска JavaScript на компьютере. Без неё ничего не работает.

Откройте **Терминал** (Ctrl+Alt+T) и введите команды по одной, нажимая Enter после каждой:

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
```
*Введите пароль своего пользователя когда спросит. Звёздочки не видны — это нормально.*

```bash
sudo apt-get install -y nodejs
```

Проверьте что установилось:
```bash
node --version
npm --version
```
Должно показать что-то вроде `v20.x.x` и `10.x.x`. Если показывает — всё ок, идём дальше.

---

### Шаг 1.2 — Установка Java (нужна для Android)

Android приложения компилируются на Java. Устанавливаем:

```bash
sudo apt-get install -y openjdk-17-jdk
```

Проверяем:
```bash
java -version
```
Должно показать `openjdk version "17..."`.

---

### Шаг 1.3 — Установка Android Studio

Android Studio — это программа для сборки Android-приложений. Это большая загрузка (~1 ГБ).

1. Откройте браузер, перейдите на: **https://developer.android.com/studio**
2. Нажмите большую кнопку **Download Android Studio**
3. Согласитесь с лицензией, начнётся загрузка файла `.tar.gz` (~1 ГБ)
4. Дождитесь окончания загрузки

После загрузки вернитесь в Терминал:

```bash
cd ~/Загрузки
```
*(Или `~/Downloads` если папка по-английски)*

```bash
tar -xzf android-studio-*.tar.gz -C ~/
```
*Это распакует Android Studio в вашу домашнюю папку. Займёт минуту.*

```bash
cd ~/android-studio/bin
./studio.sh
```
*Откроется Android Studio. При первом запуске будет долгая установка компонентов — подождите 10-15 минут.*

**Важно в Android Studio:**
- Когда спросит "Install type" — выберите **Standard**
- Дождитесь завершения всех загрузок в "SDK Setup"
- Нажмите **Finish**

---

### Шаг 1.4 — Настройка переменных окружения

Это говорит системе где находится Android. В Терминале:

```bash
echo 'export ANDROID_HOME=$HOME/Android/Sdk' >> ~/.bashrc
echo 'export PATH=$PATH:$ANDROID_HOME/tools:$ANDROID_HOME/platform-tools' >> ~/.bashrc
source ~/.bashrc
```

Проверяем:
```bash
adb --version
```
Должно показать версию `Android Debug Bridge`.

## ЧАСТЬ 2 — ПОДГОТОВКА ПАПКИ ПРОЕКТА

### Шаг 2.1 — Создаём папку проекта

```bash
mkdir -p ~/BudgetPRO
cd ~/BudgetPRO
```

### Шаг 2.2 — Скачиваем Chart.js

Chart.js нужен локально чтобы работало офлайн. Скачиваем его:

```bash
curl -L "https://cdn.jsdelivr.net/npm/chart.js@4.4.0/dist/chart.umd.min.js" -o chart.min.js
```

Проверяем что файл нормальный:
```bash
ls -lh chart.min.js
```
Должно показать что-то около `200K`. Если показывает `21 bytes` — скачалась ошибка, попробуйте ещё раз.

### Шаг 2.3 — Копируем файлы проекта

Скопируйте в папку `~/BudgetPRO` следующие файлы (скачайте из чата и перетащите):
- `index.html`
- `index.css`
- `index.js`
- `sw.js`
- `manifest.json`
- `chart.min.js` (только что скачали)
- Иконки: `favicon-96x96.png`, `apple-touch-icon.png`, `favicon.ico`, `favicon.svg`

*Если иконок нет — создайте любой PNG файл 96×96 пикселей и назовите его `favicon-96x96.png`*

Проверяем что всё на месте:
```bash
ls -la ~/BudgetPRO
```

## ЧАСТЬ 3 — УСТАНОВКА CAPACITOR

Capacitor — инструмент который превратит сайт в приложение.

```bash
cd ~/BudgetPRO
npm init -y
```
*Создаст файл `package.json` — это паспорт проекта*

```bash
npm install @capacitor/core @capacitor/cli @capacitor/android
```
*Установит Capacitor. Займёт 1-2 минуты.*

### Шаг 3.1 — Инициализируем Capacitor

```bash
npx cap init
```

Программа задаст вопросы — отвечаем так:
- `? What is the name of your app?` — напишите: **БюджетPRO** (Enter)
- `? What is the App ID?` — напишите: **pro.budget.app** (Enter)
- `? What is the web asset directory?` — напишите: **.** (просто точка, Enter)

### Шаг 3.2 — Открываем capacitor.config.json

```bash
nano capacitor.config.json
```

Сейчас там будет что-то вроде:
```json
{
  "appId": "pro.budget.app",
  "appName": "БюджетPRO",
  "webDir": "."
}
```

Нам нужно привести к такому виду. Удалите всё и вставьте:

```json
{
  "appId": "pro.budget.app",
  "appName": "БюджетPRO",
  "webDir": ".",
  "server": {
    "androidScheme": "https"
  },
  "android": {
    "allowMixedContent": true,
    "captureInput": true,
    "webContentsDebuggingEnabled": false
  },
  "plugins": {
    "SplashScreen": {
      "launchShowDuration": 1500,
      "backgroundColor": "#0d0820",
      "androidSplashResourceName": "splash",
      "showSpinner": false
    }
  }
}
```

Сохраняем: **Ctrl+O**, Enter, **Ctrl+X**

### Шаг 3.3 — Добавляем Android

```bash
npx cap add android
```

*Это создаст папку `android/` с полным Android-проектом. Займёт минуту.*

```bash
npx cap sync
```

*Копирует ваши файлы в Android-проект и синхронизирует всё.*

## ЧАСТЬ 4 — НАСТРОЙКА ANDROID ПРОЕКТА

### Шаг 4.1 — Открываем в Android Studio

```bash
npx cap open android
```

*Откроется Android Studio с вашим проектом. Первый раз будет долго индексировать — подождите пока внизу экрана не исчезнет прогресс-бар.*

### Шаг 4.2 — Разрешаем интернет в приложении

В Android Studio слева есть дерево файлов. Найдите и откройте файл:

```
app → manifests → AndroidManifest.xml
```

Двойной клик — откроется код. Найдите строку:
```xml
<manifest xmlns:android=...
```

Сразу **после** этой строки, **перед** строкой `<application`, добавьте:

```xml
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
<uses-permission android:name="android.permission.VIBRATE" />
<uses-permission android:name="android.permission.RECEIVE_BOOT_COMPLETED" />
<uses-permission android:name="android.permission.USE_BIOMETRIC" />
<uses-permission android:name="android.permission.USE_FINGERPRINT" />
```

Нажмите **Ctrl+S** для сохранения.

### Шаг 4.3 — Меняем цвета темы

Найдите файл:
```
app → res → values → styles.xml
```

Откройте, найдите строку с `colorPrimary` и замените цвет на `#a78bfa`:

```xml
<item name="colorPrimary">#a78bfa</item>
<item name="colorPrimaryDark">#0d0820</item>
<item name="colorAccent">#f472b6</item>
```

Сохраните **Ctrl+S**.

## ЧАСТЬ 5 — ТЕСТИРОВАНИЕ

### Вариант А — Тест прямо на компьютере (быстро)

Самый простой способ проверить что всё работает — запустить через браузер с локальным сервером.

В Терминале:
```bash
cd ~/BudgetPRO
npx serve .
```

Откроется ссылка типа `http://localhost:3000` — откройте её в Chrome.

В Chrome нажмите **F12** (DevTools) → вкладка **Application** → **Service Workers** — должен быть зарегистрирован ваш sw.js.

### Вариант Б — Тест на реальном Android телефоне

**Подготовка телефона:**
1. На телефоне идёте в **Настройки → О телефоне**
2. Нажмите на **Номер сборки** 7 раз подряд — появится "Режим разработчика"
3. Вернитесь в **Настройки → Для разработчиков**
4. Включите **Отладка по USB**

**Подключение:**
1. Подключите телефон к компьютеру кабелем USB
2. На телефоне появится вопрос "Разрешить отладку?" — нажмите **Разрешить**

В Терминале:
```bash
adb devices
```
Должно показать ваш телефон в списке.

**Запуск на телефоне из Android Studio:**
1. В верхней панели Android Studio выберите ваш телефон в выпадающем списке (рядом с кнопкой ▶)
2. Нажмите зелёную кнопку **▶ Run**
3. Приложение установится и запустится на телефоне автоматически

### Вариант В — Тест на эмуляторе (без телефона)

В Android Studio:
1. Меню **Tools → Device Manager**
2. Нажмите **Create Device**
3. Выберите **Pixel 6** → Next
4. Выберите **API 33** (Android 13) → Download → Next → Finish
5. Нажмите кнопку ▶ рядом с созданным устройством
6. Дождитесь запуска эмулятора (1-2 минуты)
7. Нажмите **▶ Run** в Android Studio

## ЧАСТЬ 6 — СБОРКА ФИНАЛЬНОГО APK

Когда всё работает и вы готовы поделиться приложением:

### Шаг 6.1 — Сборка debug APK (для себя, тестирование)

В Android Studio:
1. Меню **Build → Build Bundle(s) / APK(s) → Build APK(s)**
2. Подождите 1-2 минуты
3. Снизу появится уведомление "Build successful" со ссылкой **locate** — нажмите
4. Откроется папка с файлом `app-debug.apk`

Этот файл можно:
- Перенести на телефон и установить (нужно разрешить "Установку из неизвестных источников")
- Отправить другу

### Шаг 6.2 — Сборка release APK (для публикации в Google Play)

Это сложнее — нужна подпись приложения:

1. **Build → Generate Signed Bundle / APK**
2. Выберите **APK** → Next
3. Нажмите **Create new...** (создать ключ подписи)
4. Заполните:
   - **Key store path**: выберите куда сохранить (например `~/BudgetPRO/keystore.jks`)
   - **Password**: придумайте пароль (запишите! без него не обновить приложение)
   - **Alias**: `budgetpro`
   - **Key password**: тот же пароль
   - **First and Last Name**: ваше имя
   - **Country Code**: RU
5. OK → Next
6. Выберите **release** → **Finish**
7. Готовый `app-release.apk` будет в папке `android/app/release/`

## ЧАСТЬ 8 — ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### "JAVA_HOME is not set"
```bash
export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64
echo 'export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64' >> ~/.bashrc
```

### "SDK location not found"
В Android Studio: **File → Project Structure → SDK Location** → укажите путь `~/Android/Sdk`

### Приложение запускается но белый экран
Откройте Chrome на компьютере, введите в адресную строку:
```
chrome://inspect
```
Найдите своё приложение → **inspect** → посмотрите ошибки в Console

### "Gradle build failed"
```bash
cd ~/BudgetPRO/android
./gradlew clean
```
Затем снова **▶ Run** в Android Studio

### Chart.js не работает офлайн
Убедитесь что файл `chart.min.js` лежит рядом с `index.html` и его размер больше 100KB.

```
├── index.css           ← стили

├── apple-touch-icon.png

├── capacitor.config.json ← настройки Capacitor

└── android/            ← Android проект (создал Capacitor)
    └── app/

            └── app-release.apk  ← готовое приложение
```
